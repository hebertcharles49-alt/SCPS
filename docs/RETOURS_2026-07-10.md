## Verdict UI

L'UI ressemble encore à un tableau de bord pour concepteur de simulation plutôt qu'à une
interface qui enseigne le jeu. Le problème central est la HIÉRARCHIE : presque toutes les
informations ont la même importance visuelle, sont petites, et demandent un survol.

### Problèmes prioritaires
1. LISIBILITÉ/ÉCHELLE : échelle UI 100/125/150 % ; texte courant ≥16-17px ; secondaire ≥14px ;
   cibles cliquables ≥32px (idéal 40-44) ; boutons vitesse et A/V marché trop petits.
2. TOPBAR SURCHARGÉ : regrouper en 4 blocs — royaume (or, pop, stabilité) · économie
   (nourriture, prod critique, déficit) · politique (légitimité, influence, factions) ·
   temps (âge, date, vitesse). Les 5 matières → tiroir éco, SAUF pénurie réelle qui remonte
   en alerte explicite : « Fer : rupture dans 12 jours ».
3. ZONES CONCURRENTES : faire du panneau gauche la zone contextuelle UNIQUE
   (province/pays/armée sélectionné, détails en onglets). Un écran profond REMPLACE ou
   réduit le panneau contextuel, il ne s'y ajoute pas.
4. ACTIONS INDIFFÉRENCIÉES : primaire = or clair, plus grande · secondaire = neutre ·
   destructive = rouge sombre + CONFIRMATION · navigation = léger. Avant une décision :
   coût, durée, effets immédiats, conséquences probables, raison d'indispo.
   Ex : « Purger — −1 200 habitants · satisfaction −35 · coercition +12 · risque diplo élevé ».
5. CLAVIER SACRIFIÉ : focus_mode=NONE global (ui_theme.gd:90) supprime Tab/focus/Entrée.
   Garder les raccourcis globaux MAIS : Tab fonctionnel, focus visible, Entrée valide,
   Échap ferme, Espace intercepté globalement sans supprimer le focus, raccourcis réaffectables.
6. COULEUR SEULE : sense() rouge ≈ 3.6:1 — insuffisant. Doubler d'un second canal :
   ▲ ▼ ! cadenas contour, mots (« déficit », « stable »), formes dans les graphiques.
7. PORTRAITS CONSEILLERS non incorporés : un portrait = une identité = un modificateur
   léger trouvable dans tunes ; idem agent héritage, agent tech, agent event.

### Par écran
- Menu : ajouter une phrase de proposition (« Façonnez un peuple et observez ses
  institutions transformer le monde »). Nouvelle partie : combler le vide (résumé du monde).
- Créateur : héritages/éthos en CARTES (identité visuelle, style de jeu, atout, faiblesse,
  conséquence narrative). État sélectionné plus marqué. « Passer » ambigu.
- Carte : fog immense noir → fond parchemin cartographique plus clair / motifs inconnus.
  La carte doit rester la zone la plus lumineuse.
- Province : SYNTHÈSE en haut (problème principal · opportunité · action recommandée,
  ex « ⚠ Logements presque saturés — construire avant +1 000 habitants »). Disques
  Culture/Idéologie → barres compactes.
- Éco/stocks/marché : les plus illisibles. Trier pénuries d'abord ; nom de ressource sur
  les lignes problématiques ; A/V → Acheter/Vendre.
- Diplo/conseil : états vides expliqués (« Aucun pays étranger connu — explorez… »).
  Conseil : portraits, traits, loyauté, effet attendu.
- Tech : nom au survol rapide, panneau latéral de détail, coût/durée, prérequis manquant,
  bouton de recherche explicite, couleur+symbole. Métabolisation → panneau séparé.
- Construction : ajouter l'EFFET (que produit-il, quel problème résout-il, quand fini) +
  filtres (disponible/abordable/militaire/économie/administration).
- Codex : organiser par INTENTION (développer une province, résoudre une pénurie, gouverner
  les cultures, guerre, commerce, éviter une crise) + recherche textuelle.
- Annales : vraie chronologie (années, icônes, lieux cliquables, conséquences).

## Passe éditoriale TECH — principes
- Hover mécanique court, chiffré, non ambigu ; flavor = ligne distincte en italique.
- dF/dEco/dMil = champs MORTS (jamais lus) : ne JAMAIS les promettre dans l'UI.
- NODE_PROD_PCT et NODE_EFF_PCT s'additionnent dans le même tech_prod GLOBAL : une tech
  « agricole » améliore TOUTE la production — le hover doit être honnête sur la portée.
- Techs civiles sûres : pas de petite charge cachée (0.2-0.3) — la retirer ou l'afficher.
- Branches : Savoir = recherche+K+arcane (PAS la meilleure branche de prod en plus) ;
  Forge = prod+dégâts (+risques) ; Société = L+cohésion+moral−fracture.
- Budget par palier : T1 5-8 % · T2 8-10 % · T3 10-15 % · T4 doctrine 20 % (éco 8-12 %) ·
  T5 signature ciblée (+50 % arquebusiers).
- Chaîne Scriptorium→Académie→Université : RETIRER le +eff% (garde recherche + K).
- Séparer déverrouillage et bonus numérique (Poudrière/Comptoirs/Halles/Invocation
  n'ont pas besoin d'un second bonus).

### TECH — changements d'effets demandés (résumé opérationnel)
- TECH_SCRIPTORIUM/ACADEMIE/UNIVERSITE : retirer NODE_EFF_PCT (+5/+7/+10 %) ; garder
  recherche +50 % et dK 1/2/3.
- TECH_WARDS : retirer charge=0.3 (ou l'afficher) ; garder dL=0.5.
- TECH_SCRYING : +0.5 K (renforcer l'identité) ; garder puissance 0.5, flux 0.3, charge 1.
- TECH_OUTILLAGE : flux/charge → 0 (étape industrielle sûre) ; garder +10 % prod.
- TECH_MANUFACTURE : hover corrigé — retirer « +puissance brute » (dPuissance=0).
- TECH_QUALITE_MATERIAUX : AJOUTER +5 % prod (NODE_PROD_PCT) ; renommer « Taille de précision ».
- TECH_FORTIFICATIONS : retirer charge=0.2 ; hover sans dF/dMil morts.
- TECH_AUTOMATES : AJOUTER +8 % prod (NODE_PROD_PCT).
- TECH_ARMURERIE : retirer charge=0.3 ou l'annoncer.
- TECH_ALCHIMIE : AJOUTER +5 % prod (NODE_PROD_PCT).
- Renommages (7) : Scrying→Clairvoyance · Collecte de bois→Sylviculture · Collecte
  d'argile→Extraction d'argile · Collecte de nourriture→Subsistances · Qualité des
  matériaux→Taille de précision · Mécanisme d'horlogerie→Horlogerie civique ·
  Foederati→Fédérés. (+ Savoir de guerre = « Collège de guerre » en hover, Magie de
  bataille = « Tour de mages », etc. — voir flavors.)
- Rouages/Horlogerie/Glyphes/Communion éthérée : annoncer les petits flux/charges dans le
  hover (ou les retirer pour Rouages si trop mineur — préférence : ANNONCER).
- Combos « agricoles » (Druide/Charrues/Machines agricoles/Abondance/Irrigation/Vergers/
  Pâturages) : hover honnête « production globale » tant qu'aucun canal agricole dédié.
- Académie cosmopolite/Concile : « efficacité générale », pas « recherche » seule.

## Passe éditoriale ÉVÉNEMENTS — principes
- Chaque événement gagne une OUVERTURE (courte scène 1-2 phrases AVANT les choix).
- Chaque option gagne un FLAVOR (la voix du souverain/conseiller qui choisit).
- Les EFFETS restent INCHANGÉS (display-only).
- Titres renommés proposés (Quake→« La nuit où la terre cria », etc.).
- Pas de tirets longs. %s préservés exactement.
